# 변경사항 v109 — 홍보 원고 생성기 + 리드 인입 보강

배포본: `2026-07-19-ad-draft-v109`

## 1. ✍️ 홍보 원고 생성기 (광고 관리 탭)

**무엇**: [📣 광고] 탭의 각 채널(스레드·인스타·메타·네이버 블로그)에
**[✍️ 원고 만들기]** 버튼이 생겼습니다. 현장을 고르면 그 현장의 자료
(지역·평수·공정·일정)를 읽어 채널에 맞는 홍보 원고를 즉석에서 써 줍니다.

- **네이버 블로그**: "대전 [동네] [평수]평 인테리어 후기 — [공정]" 제목 공식의
  본문 초안(개요 표·공정별 항목·사진 자리·연락처). 이 제목 형식은 공개 사이트
  콘텐츠 전략(CONTENT-PLAN)에서 실측한, 지역 검색 상위 노출 형식입니다.
- **스레드·인스타**: 짧은 캡션 + 자동 해시태그
  (#대전인테리어 #[구]인테리어 #[동] #욕실리모델링 등 공정·지역 기반)
- **메타(페북)**: 중간 길이 소개문

동작 원칙:
- 외부 API 없음 — 무료 규칙 기반, 오프라인에서도 동작
- 고객 실명·금액은 원고에 넣지 않음 (대괄호 [ ] 자리만 직접 채우면 완성)
- [📋 복사] → 채널에 붙여넣기, [게시 기록에 추가] → 기존 adPosts 기록으로 저장
- 현장 데이터 출처: 현장명/고객주소(지역·평수), phases(공정),
  일정표 날짜 범위(기간), 홈페이지 리드 노트(평수·희망공사 폴백)

## 2. 광고 탭 빈화면 버그 수정

파일이 0개면 광고 탭이 "파일을 끌어다 놓으세요" 빈 상태로 대체되던 문제.
광고 관리는 파일과 무관하므로 예외 목록에 추가 — 폴더 연결 전에도 열립니다.

## 3. 리드 인입 hashchange 보강

앱이 **이미 열려 있는 탭**에서 `#lead=` 딥링크로 재진입(주소만 변경, 재로드
없음)하면 상담 인입이 트리거되지 않던 틈을 hashchange 리스너로 보강.
(기존 새 탭 열기 경로는 원래대로 동작)
