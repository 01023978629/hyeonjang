# 변경사항 v105 — 모바일 클라우드 저장 (Apps Script 중계)

날짜: 2026-07-17 · 기준: v104(= 저장소 v103 캐시 + 리드 수신부) → **v105**
원본 백업: `backup/index_v104_original.html`

## 왜 바꿨나

PC에서는 Google Drive 읽기·쓰기가 됐지만, **모바일 사파리·카톡/네이버 인앱 브라우저**에서는
Google OAuth 팝업·토큰 문제로 저장이 되지 않았다 (`cloudAutoSave()`가 `__gdToken` 없으면 종료).
모바일에서 보이던 데이터도 실제로는 IndexedDB 캐시였다.

## 새 구조

```
모바일·PC 웹 → 일반 HTTPS(text/plain POST) → Apps Script 웹앱(대표 계정 권한으로 실행)
             → 기존 Google Drive '만물인테리어' 폴더 / 현장데이터.json  (데이터 이전 없음)
```

- 모바일에서 **Google 로그인 팝업·토큰 없이** 저장·불러오기·사진/문서 업로드 가능
- 기존 Google 직접 연동(gd*)은 **전부 보존** — 설정 모달의 "비상용" 접이식 섹션

## 동작 변화 (기본 경로)

| 상황 | v104 | v105 |
|---|---|---|
| 화면 수정 | 로컬(IndexedDB)만, gd 토큰 있을 때만 Drive | 로컬 즉시 + **3초 후 서버(relay) 저장** (relay 설정 시) |
| 모바일 저장 | ❌ (OAuth 팝업 불가) | ✅ 로그인 없이 저장 |
| 동시 수정 | 마지막 저장이 조용히 이김 | **revision 충돌 감지** → 3택 모달 |
| 오프라인 | 로컬만, 유실 가능성 인지 불가 | 큐 등록 + "전송 대기 N건" 표시 → 온라인 복귀 시 자동 재전송 |
| 상태 표시 | ☁️ 저장됨(성공 시만) | 기기 저장/클라우드 저장 중·완료/오프라인 대기/충돌/서버 연결 실패/인증키 오류 |
| 사진 업로드 | 원본 그대로(OAuth 필요) | 압축(장변 1600px JPEG) 후 로그인 없이, 실패는 재시도 큐 |

relay 미설정 시: v104와 동일하게 동작(로컬 + gd 토큰 경로). **설정 전까지 아무것도 바뀌지 않음.**

## 데이터 손실 방지 장치

1. 서버 자료를 적용하는 모든 경로(불러오기·충돌 해결·최신 배너·첫 연결) 직전 `hjSnapshot(..., true)` 강제
   — **스냅샷이 실패하면 적용 자체를 중단**(조용히 덮어쓰지 않음, relayGuardSnapshot)
2. revision 불일치 시 서버가 절대 덮어쓰지 않고 conflict 반환 → 사용자 3택
   (③ 라벨은 실제 동작 그대로: "내 기기 자료를 이 기기 안전판에 백업한 뒤 서버 자료 적용")
3. 첫 연결: 양쪽 자료 비교 안내 + 선택 전 자동 백업 — 서버측 백업 실패 시 **확인 모달**(취소가 기본)
4. 하루 1회 서버 날짜별 백업(보관 14개, 초과 자동 정리) — 큐 경유 저장에도 적용
5. 오프라인 큐(저장은 최신 1건 병합, 업로드는 최대 15건 상한) + 5회 실패 시 수동 재전송,
   설정에서 [실패 항목 비우기] 가능. 재전송 도중 새로 실패한 항목이 유실되지 않도록 id 기준 정리
6. 부팅 시 서버가 최신이어도 자동 적용하지 않음 — 배너 [불러오기]는 **클릭 시점에 재조회** 후 적용
7. 촬영 사진의 Drive 파일 연결(_driveId)은 relay 경로에서도 레거시와 동일하게 유지

## 보안상 남은 한계 (정직 고지)

- APP_TOKEN은 브라우저(IndexedDB)에 저장 → **완전한 비밀이 아님**. 1인 내부 업무용 최소 인증.
  다중 사용자 운영 시 Supabase Auth 등 계정 기반 인증으로 교체 필요.
- Apps Script 실행 한계로 대용량 영상·원본 파일 업로드는 미지원(UI에 표기).
- 기존 index.html의 GD_CLIENT_ID·GD_FOLDER_ID 하드코딩은 비상용 경로와 함께 유지
  (신규 코드에는 하드코딩 없음).

## 사용자가 직접 해야 하는 것

1. `APPS_SCRIPT_설치방법.md` 대로 Apps Script 배포 (구글 계정 권한 승인 포함, 15~20분)
2. 웹 설정 화면에 서버 URL + APP_TOKEN 입력 → 연결 테스트 → 첫 연결 안내에서 자료 선택
3. 실기기 검증: `모바일_클라우드저장_검증체크리스트.md`
