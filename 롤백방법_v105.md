# 롤백 방법 v105 → v104

v105(클라우드 중계)에서 문제가 생겼을 때 되돌리는 방법. 쉬운 것부터 순서대로.

## 0단계 — 롤백 전 확인

- v105는 **relay 설정을 입력하기 전까지 v104와 동일하게 동작**합니다.
  문제가 relay 저장에서만 난다면 롤백 대신 1단계로 충분합니다.
- 데이터 자체는 Drive `현장데이터.json` + 날짜별 백업 + 앱 내 스냅샷(안전판)에 있어
  코드 롤백으로 데이터가 사라지지 않습니다.

## 1단계 — 코드 롤백 없이 끄기 (가장 안전, 1분)

1. 앱 ☁️ 설정 → "권장: 서버 연결" 칸의 **서버 주소를 지우고 저장**
2. 자동 저장이 즉시 v104 방식(로컬 + 비상용 Google 직접 연동)으로 복귀
3. 필요하면 비상용 섹션에서 기존 Google 로그인으로 저장/불러오기

## 2단계 — 파일 롤백 (git 없이, 5분)

1. 저장소의 `backup/index_v104_original.html`을 내려받아 `index.html`로 교체
2. `sw.js`의 `const C='hyeonjang-v105'`를 `'hyeonjang-v106-rollback'` 등 **새 값으로 변경**
   (⚠️ v103으로 되돌리면 기존 캐시와 충돌 — 반드시 새 키로)
3. main에 커밋·푸시 → GitHub Pages 자동 배포
4. 기기에서 앱 새로고침 2회(서비스워커 교체)

## 3단계 — git 롤백 (정석)

```bash
git revert <v105 머지 커밋>   # 또는:
git checkout main && git revert --no-edit <머지커밋SHA>
git push origin main
```

- 머지 커밋 SHA는 GitHub의 PR 페이지(제목 "모바일 클라우드 저장 v105")에서 확인
- revert 후에도 2단계의 sw.js 캐시 키 새 값 규칙 적용 확인

## 데이터 되돌리기 (코드와 별개)

- 서버(relay)가 잘못된 데이터를 저장했다면: Drive `만물인테리어` 폴더의
  `현장데이터_백업_YYYY-MM-DD.json`을 열어 내용 확인 → 앱의 "⏮ 이전 버전 복원"
  또는 파일 내용을 `현장데이터.json`에 복사
- 기기 쪽 데이터: 앱 안전판(스냅샷) 목록에서 해당 시점("충돌-서버적용 전",
  "서버 첫연결 전" 등 라벨)으로 복원

## Apps Script 서버 정리 (선택)

- 완전히 걷어내려면: script.google.com → 해당 프로젝트 → 배포 관리 → 배포 보관처리
- APP_TOKEN 노출이 의심되면 스크립트 속성에서 값만 교체해도 즉시 차단됨
