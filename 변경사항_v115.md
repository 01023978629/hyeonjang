# 변경사항 v115 — 사진 "올리기" 구글 로그인 없이도 되게 (P0 수정)

배포본: `2026-07-19-photo-local-v115`

## 문제 (대표님 리포트)
사진이 안 올라감. 원인:
- 폰의 [올리기] 버튼(gdUploadPhotos)이 **릴레이 미설정 시 구글 드라이브
  로그인을 강제**했는데,
- 아이폰에서 팝업 차단 → '이동 방식 로그인' → 구글이
  **400 redirect_uri_mismatch** 로 거부(콘솔에 이 주소가 승인 URI로
  미등록). 결국 사진 자체가 안 올라갔다.
- (촬영 버튼은 원래 로컬 저장 우선이라 정상이었음)

## 조치
1. [올리기]를 촬영과 동일하게 **폰에 먼저 저장(ingestFile)** → 구글
   로그인 없이 항상 저장·미리보기·현장 자동배정. 클라우드(드라이브)는
   이미 로그인돼 있을 때만 백그라운드 백업, 없으면 로그인 강요 안 함.
   릴레이 설정 시엔 기존대로 릴레이 업로드(변경 없음).
2. '구글 로그인 실패' 모달에 안내 추가: "사진은 이미 폰에 저장됩니다"
   + [☁️ 클라우드 저장 설정(구글 없이)] 버튼 → 깨진 redirect 로그인을
   반복하지 않고 중계(릴레이) 경로로 유도.

## 근본 해결(권장)
여러 기기에서 사진을 함께 보려면 **구글 없이 되는 클라우드 저장(중계,
Apps Script 릴레이)** 을 연결하세요(더보기 → ☁️). 구글 드라이브를
계속 쓰려면 Google Cloud 콘솔의 OAuth 클라이언트 '승인된 리디렉션 URI'에
이 앱 주소(https://01023978629.github.io/hyeonjang/)를 등록해야 합니다.
