## 1. 한눈에 보기

만물인테리어(대전 1인 인테리어) 앱은 단순 기록 앱에서 출발해, **AI가 사업을 매일 점검하고 → 할 일을 만들고 → 안전한 건 스스로 처리하고 → 위험한 건 승인받고 → 결과를 기록하고 → 다음으로 잇는** 자율 운영 시스템으로 완성됐습니다.

핵심은 **닫힌 루프(closed loop)**입니다. 한 번 실행하고 끝나는 게 아니라, 실행 결과를 보고 다음 작업을 스스로 만들어 순환합니다.

---

## 2. 자율 운영 루프 구조 (실제 코드 연결)

```
  아침 부팅 / 앱 복귀 / 1시간 간격
        ↓
  ① Daily·Weekly·Monthly Scan   (aiPlannerScan)
        ↓
  ② Planner — 9개 영역 감지       (미수금·일정·작업일지·사진·견적·예산·AS·주간·월말)
        ↓
  ③ Queue — 중복제거·우선순위     (aiQueueEnqueue / urgent·high·normal·low)
        ↓
  ④ Approval — 승인/배치승인      (aiQueueApprove / aiQueueApproveBatch)
        ↓
  ⑤ Executor — 안전작업 자동실행   (aiOpsAutoExecuteSafe: 조회·리포트)
              위험작업 승인 후 실행  (aiExecuteTask: 문자·수금)
        ↓
  ⑥ 후속 체이닝 — 결과 확인 작업 자동 생성  (aiOpsChainNext)
        ↓
  ⑦ Log — 전 과정 기록            (aiOpsLog, 최근 300건)
        ↓
  ⑧ 자가진단 — 성숙도 점수         (aiOpsSelfCheck: 0~100점)
        ↓
  ⑨ Next Check — 다음 점검 예약    (aiScheduleNextCheck)
        ↓
     [ 다시 순환 ]
```

**AI 브레인(aiOpsBrain):** 규칙 스캔과 별개로, AI가 사업 상태 전체를 보고 "오늘 먼저 할 일 3가지 + 이유"를 추론.

**오토파일럿(aiAutoOperate):** 한 번 누르면 위 전 과정을 실행하고 "자동 처리한 일 + 승인 필요한 일"을 통합 리포트로 제시.

## 4. 안전장치 (되돌리기 어려운 작업 보호)

- **문자 발송·수금 처리·삭제·일정 변경·견적 생성은 항상 사용자 승인 후 실행.** 코드에 `requiresApproval` 플래그로 강제되어, 자율 실행 대상에서 원천 차단.
- **자동 실행 대상은 `readOnly && action.kind==='tool'`만** (조회·리포트). 화면 여는 작업(open)이나 문자(sms)는 자동 실행 안 함.
- 모든 저장 전 `hjSnapshot()` 스냅샷, 전 과정 `try/catch`.
- 큐 중복 방지(source 기준) + 재부팅 시 중복 폭발 방지(오늘 점검 여부 키).
- 빈 데이터(첫 사용자)에서도 전체 루프 무오류 동작 확인.

## 6. 남은 한계 (정직한 고지)

1. **앱이 완전히 닫혀 있으면 백그라운드 자동 실행 불가.** 웹/PWA 제약. 앱 실행·복귀·visibilitychange·setInterval 기반으로만 동작하며, 앱을 열 때 밀린 점검을 수행합니다. (알림 권한 허용 시, 복귀 시점에 긴급 대기 알림으로 호출.)
2. **문자·수금의 최종 실행은 사람이 확정.** 이건 한계가 아니라 **안전장치**입니다. 되돌리기 어려운 일은 사장님이 쥐고 있는 게 맞습니다.
3. **AI 브레인은 외부 AI(Gemini/ChatGPT 등) 키 설정 필요.** 미설정 시 규칙 기반 루프는 정상 작동, 브레인만 안내 후 비활성.
4. **Planner 감지는 입력 데이터에 의존.** 연락처·완공일·원가가 비어 있으면 놓칠 수 있어, 🧯 데이터 건강 검진과 병행 권장.
