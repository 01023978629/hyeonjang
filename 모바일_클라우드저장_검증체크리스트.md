# 모바일 클라우드 저장 검증 체크리스트 (v105 실기기)

Apps Script 배포(`APPS_SCRIPT_설치방법.md`) 완료 후, 실제 기기에서 아래를 순서대로.
**모든 테스트는 가짜 데이터로** (실제 고객 정보 입력 금지).

## A. 연결 (PC 크롬)

- [ ] ☁️ 설정 → 서버 주소·인증키 입력 → **연결 테스트** 초록(폴더 OK, revision 표시)
- [ ] 첫 연결 안내 모달: 서버/기기 자료 비교가 뜨고, 선택 전 백업 문구 확인
- [ ] 인증키 일부러 틀리게 → "인증키 오류" 표시 (앱 멈춤 없음)
- [ ] 서버 주소 비우면 → 이전(v104)처럼 로컬 저장만, 오류 없음

## B. 저장·동기화 (PC ↔ 폰)

- [ ] PC: 테스트 현장 1개 추가 → 3초 내 "클라우드 저장 완료" 표시
- [ ] Drive 폴더에서 `현장데이터.json` 수정시각 갱신 확인
- [ ] 폰(사파리): 앱 열기 → "서버에 더 최신 자료" 배너 → [불러오기] → PC에서 만든 현장 보임
- [ ] 폰: 메모 수정 → 저장 완료 표시 → PC 새로고침 배너 → 불러오기 → 폰 수정 반영
- [ ] **로그인 팝업이 한 번도 뜨지 않았는지** ← 이번 버전의 핵심

## C. 인앱 브라우저 (카톡·네이버)

- [ ] 카톡 대화방에 앱 주소 보내고 카톡 안에서 열기 → 데이터 보임 → 수정 → 저장 완료 표시
- [ ] 네이버 앱 검색으로 열기 → 동일 확인
- [ ] (선택) 카톡 안에서 사진 1장 업로드

## D. 오프라인·충돌

- [ ] 폰 비행기 모드 → 수정 → "오프라인, 전송 대기 1건" → 비행기 모드 해제 → 자동 재전송 → 대기 0건
- [ ] 충돌: PC와 폰 둘 다 연 상태에서 PC 수정·저장 → 폰에서 **불러오기 없이** 바로 수정·저장
      → 충돌 모달 3택 표시 → "서버 데이터 사용" 선택 → 폰 수정분이 스냅샷(안전판)에 남아 있는지 확인
- [ ] 같은 충돌에서 "내 기기 데이터로 덮어쓰기" → PC에서 불러오면 폰 내용

## E. 사진·문서 (폰)

- [ ] 카메라 촬영 사진 1장 업로드 → Drive `현장사진` 폴더에 생성, **세로 사진이 눕지 않는지**(EXIF)
- [ ] 갤러리에서 3장 연속 업로드 → 진행 표시(n/3) → 3장 모두 도착, 용량이 원본보다 작아짐(압축)
- [ ] 아이폰 HEIC 사진 1장 → 업로드 성공 여부 기록 (실패 시 원문 캡처)
- [ ] PDF 1개 업로드 → `견적서` 폴더 도착
- [ ] 동영상/대용량 파일 시도 → "지원하지 않음" 안내가 뜨는지

## F. 백업·복구

- [ ] 저장 다음 날(또는 서버에서 backup 호출) → Drive에 `현장데이터_백업_날짜.json` 생성
- [ ] 같은 날 두 번째 백업 시도 → 중복 생성 안 됨
- [ ] 앱 "⏮ 이전 버전 복원" 목록이 여전히 동작 (기존 기능 회귀 확인)

## G. 기존(비상용) 경로 회귀

- [ ] ☁️ 설정의 "비상용: Google 직접 연동" 펼침 → PC에서 기존 로그인 → 기존 저장/불러오기 동작
- [ ] 서버 주소 지운 상태에서 기존 자동 저장(gd 토큰) 경로 동작

## H. PWA 갱신

- [ ] 기존 홈화면 설치 앱: 두 번 새로고침 후 새 버전(설정에 서버 연결 칸 존재) 확인
